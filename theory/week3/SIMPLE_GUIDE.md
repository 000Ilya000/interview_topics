## SSR / SSG / ISR — «когда готовить HTML»

| | Когда HTML готов | Жизненная аналогия |
|---|------------------|-------------------|
| **CSR** | В браузере пользователя | Блюдо готовишь дома после заказа |
| **SSR** | На сервере при каждом запросе | Готовят когда ты пришёл в ресторан |
| **SSG** | При сборке проекта | Заготовки с утра, разогревают |
| **ISR** | SSG + обновление раз в N минут | Меню обновляют каждый час |

**Когда SSR вредит perf?**  
Много запросов + тяжёлый сервер + персональные данные = каждый ждёт готовку.  
*Лучше:* статика на CDN + client fetch для личного кабинета.

**Hydration** — React «оживляет» готовый HTML. Если server и client нарисовали разное — баг.

## Server vs Client Components

- **Server** — грузит данные на сервере, **0 JS** на клиент для этого куска. Нельзя `onClick`, `useState`.
- **Client** — `'use client'`, hooks, клики.

*Жизнь:* официант (Server) приносит еду. Ты (Client) ешь и жмёшь кнопку «ещё соус».

## HTTP + CORS

**CORS** — браузер блокирует запрос с `mysite.com` на `api.other.com`, если тот API не сказал «разрешаю».

*Жизнь:* охранник на входе — «есть ли ты в списке гостей?» Preflight — звонок заранее: «можно POST с JSON?»

**JWT в cookie vs localStorage:**
- localStorage — JS читает → при XSS украдут
- httpOnly cookie — JS **не** читает → XSS не достанет

## XSS — «чужой скрипт на твоём сайте»

*Жизнь:* кто-то вклеил листовку с инструкцией «переведи деньги» прямо в твою газету. Читатель думает — это от редакции.

**Защита:** React экранирует `{text}`, не `innerHTML`, httpOnly cookie, CSP.

→ Подробно: `xssCsrf.md`

## CSRF — «сайт-обманщик от твоего имени»

Ты залогинен в банке. Открыл evil.com — там скрытая форма «переведи 1000₽». Браузер **сам** прикрепит cookie банка.

*Жизнь:* подпись уже на столе, мошенник подсовывает договор.

**Защита:** SameSite cookie, CSRF-токен, не GET для переводов.

→ Подробно: `xssCsrf.md`

## Race condition — «кто последний — тот и на экране»

Быстро кликнул «Москва» → «Питер». Питер ответил быстрее. Потом пришла Москва и **перезаписала** Питер.

**Fix:** `AbortController` — отмени старый запрос при новом клике.

→ Подробно: `reliableDataFetching.md`

## Web Vitals — «Google оценивает UX»

- **LCP** — как быстро появился главный контент (< 2.5 сек)
- **INP** — как быстро откликается на клик (< 200 мс)
- **CLS** — прыгает ли вёрстка (< 0.1)

*Жизнь:* LCP — «когда принесли основное блюдо». CLS — «стол дёрнулся когда поставили салат».
